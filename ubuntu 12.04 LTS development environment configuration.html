系统安装

从windows安装，这里是以XP为例

grub4dos安装

下载grub4dos，解压后将 grldr，menu.lst，grldr.mbr，grub.exe 放到系统盘根目录

iso文件准备

下载 ubuntu12.04的iso文件，将iso放到系统盘跟目录，并将其中的

  • casper文件夹下面的vmlinuz和initrd.lz
  • .disk文件夹

提取到系统盘跟目录。然后修改menu.lst，在其末尾增加

title Install Ubuntu12.04
root (hd0,0)
kernel (hd0,0)/vmlinuz boot=casper iso-scan/filename=/ubuntu-12.04-desktop-i386.iso locale=zh_CN.UTF-8
initrd (hd0,0)/initrd.lz

hd(x,y) 表示第x块硬盘的第y个分区，x、y都是从0开始，例如： 只有一块硬盘，主分区在C盘，则用hd(0, 0)表示

修改系统盘跟目录下的系统隐藏文件boot.ini，在文件最后添加如下内容

c:\grldr=”Ubuntu Install”

以增加grub启动

安装注意事项

通过grub引导进入ubuntu安装界面后，注意下列以下事项： + 断网，避免安装过程中去网络下载导致安装非常久。 + 同时卸载isodevice：

$ sudo umount -l /isodevice

java环境

从 oracle java下载地址 下载自己系统所需版本，解压。解压目录记作 JAVA_PATH 在/etc/profile文件末尾增加一下内容：

export JAVA_HOME=JAVA_PATH
export JRE_HOME=JAVA_PATH/jre
export CLASSPATH=.:$JAVA_HOME/lib/dt.jar:$JAVA_HOME/lib/tools.jar:$JAVA_HOME/lib:$JRE_HOME/lib:$CLASSPATH
export PATH=$JAVA_HOME/bin:$PATH

上面的JAVA_PATH需要修改为刚才jdk的解压路径。执行：

$ source /etc/profile

让刚才的配置生效。然后执行以下命令进行配置：

$ sudo update-alternatives --install "/usr/bin/java" "java" "JAVA_PATH/bin/java" 300
$ sudo update-alternatives --install "/usr/bin/javac" "javac" "JAVA_PATH/bin/javac" 300
$ sudo update-alternatives --install "/usr/bin/javaws" "javaws" "JAVA_PATH/bin/javaws" 300
$ sudo update-alternatives --config java
$ sudo update-alternatives --config javac
$ sudo update-alternatives --config javaws

同样，需要将上面的JAVA_PATH需要修改为刚才jdk的解压路径。

执行

$ java -version

查看java环境是否安装成功。

spawn-fcgi -a 127.0.0.1 -p 9000 -C 10 -u www-data -f /usr/bin/php-cgi

下面是nginx支持虚拟主机的一份简要配置：

# workspace，使用localhost
server {
    listen 127.0.0.1:80;

    root /home/tom/workspace;
    index index.php index.html index.htm;
    server_name localhost;
    autoindex on;
    autoindex_exact_size off;
    autoindex_localtime  on;

    location / {
            try_files $uri $uri/ /index.html;
    }

    location /doc/ {
            alias /usr/share/doc/;
            autoindex on;
            allow 127.0.0.1;
            deny all;
    }
    location ~ \.php$ {
            include /etc/nginx/fastcgi_params; #需放在第一行，否则会出错
            fastcgi_pass 127.0.0.1:9000;
            fastcgi_index index.php;
            fastcgi_param SCRIPT_FILENAME /home/tom/workspace$fastcgi_script_name;
    }
}

# 虚拟主机：greasemonkey
server {
        listen 127.0.0.1:80;

        root /home/tom/workspace/gm_sources;
        index index.html index.htm;
        server_name gm.me;
        autoindex on;
        autoindex_exact_size off;
        autoindex_localtime  on;

        location / {
                try_files $uri $uri/ /index.html;
        }

        location /doc/ {
                alias /usr/share/doc/;
                autoindex on;
                allow 127.0.0.1;
                deny all;
        }
}

node环境

使用nvm来版本管理node：

$ git clone git://github.com/creationix/nvm.git ~/.nvm

加入环境变量，并刷新：

$ echo '. $HOME/.nvm/nvm.sh' >> ~/.bashrc 
$ . ~/.bashrc

查看node可用版本：

$ nvm ls-remote

比如当前最新版本为 v0.11.11，进行安装：

$ nvm install v0.11.11

nvm其他常用使用为：

$ nvm use v0.11.11 # 使用0.11.11为当前shell的node版本，对其他shell是无效的
$ nvm alias default v0.11.11 # 将v0.11.11设置为默认的node版本
$ nvm ls # 查看当前系统安装的node所有版本

node 一些常用全局插件

$ npm install grunt-cli -g
$ npm install tpl2js

python环境

使用pythonbrew管理python：

$ curl -kL http://xrl.us/pythonbrewinstall | bash

pythonbrew就会安装到 ~/.pythonbrew，然后设置环境变量： 在~/.bashhrc 文件追加下面这一行

[[ -s $HOME/.pythonbrew/etc/bashrc ]] && source $HOME/.pythonbrew/etc/bashrc

查看可用python版本：

$ pythonbrew list -k

安装某个版本：

$ pythonbrew install 2.7.5

pythonbrew其他常用命令：

$ pythonbrew switch 2.7.2 # 使用某个版本的python，多个shell之间永久存在
$ pythonbrew use 2.7.2 # 仅在当前shell中使用某个版本的python
$ pythonbrew list # 罗列系统中已经安装的python
$ pythonbrew uninstall 2.7.2 # 卸载某个版本的python
$ pythonbrew update --master # 升级pythonbrew到最新版本

在pythonbrew使用virtualenv：

$ pythonbrew venv init # 初始化安装virtualenv
$ pythonbrew venv create proj # 创建某个隔离环境
$ pythonbrew venv list # 罗列当前创建的所有隔离环境
$ pythonbrew venv use proj # 进入并使用某个隔离环境
$ pythonbrew venv delete proj # 删除某个隔离环境
$ pythonbrew venv rename proj proj2 # 重命名某个隔离环境

ruby环境

使用rvm管理ruby：

$ curl -L get.rvm.io | bash -s stable
$ source ~/.bashrc
$ source ~/.bash_profile

修改 RVM 的 Ruby 安装源到国内的 淘宝镜像服务器，这样能提高安装速度

$ sed -i -e 's/ftp\.ruby-lang\.org\/pub\/ruby/ruby\.taobao\.org\/mirrors\/ruby/g' ~/.rvm/config/db

列出已知的ruby版本

$ rvm list known

安装某个版本，例如：

$ rvm install 2.1.0

如果 rvm use 出现错误，提示 RVM is not a function，可以尝试将下面这行添加到 .bashrc 中

[[ -s "$HOME/.rvm/scripts/rvm" ]] && source "$HOME/.rvm/scripts/rvm" # Load RVM into a shell session *as a function*

更多的详细信息可以参考 手把手安装RVM以及为什么RVM is not a function

rvm其他常用命令：

$ rvm use 1.9.3 # 使用一个ruby版本
$ rvm use 1.9.3 --default  # 设置为默认版本
$ rvm list # 查询已经安装的ruby
$ rvm remove 1.9.2 # 卸载一个已安装版本

安装sass：

$ gem install sass
